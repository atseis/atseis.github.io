功能

  1. 收集数据
  2. 处理数据
  3. 可视化展示
  4. 数据分析&报警处理

提供了一整套可行的解决方案，并且形成了一整个生态，能够极大地降低我们的研发成本

Since Prometheus exposes data in the same manner about itself, it can also scrape and monitor its own health.

  • Prometheus 可以监控自己本身

入门

按照官网教程进行即可

详细的操作需要参见具体的文档：

  • configuration
  • query的表达式expression language

这里只简单讲下从入门网站可以获知的内容

主要的流程也就是：

  1. 启动: prometheus --web.enable-lifecycle --config.file=prometheus.yml
  2. 然后在端口服务处试验 query 命令即可
    • 可以显示为 表格
    • 可以显示为 Graph
  3. 然后规则介绍了一个简单的功能：可以将命令“alias”，比如：
    • 将命令 avg by (job, instance, mode) (rate(node_cpu_seconds_total[5m]))
    • 记录为 job_instance_mode:node_cpu_seconds:avg_rate5m
  4. 之后重启服务后，就可以直接以记录结果代替命令进行 query

遇到的问题

Prometheus v2 启动时没有加 --web.enable-lifecycle，不好 quit, reload

  1. kill 完立即有一个新的 PID
  2. 无法通过 curl -X POST http://localhost:9090/-/quit 来退出或类似的reload
  3. 重启

猜想原因：

  1. 之前启动时没有正常结束程序，程序还在进行直接关机了

重新安装后就可以了

但是重新安装还是太麻烦，对于这种情况有没有更好的解决方法？(事后得到了大佬的解答，不过可惜没法验证了)

  • 考虑使用 systemctl disable --now prometheus 禁用
  • 状态为 enabled 的服务会自动启动（& systemd 服务可以设定异常退出后自动重启）

这里涉及到 systemd 的使用：

  • kill 和 systemd 的区别：
    • kill 只是杀掉进程，而 systemctl 是控制服务状态
    • 如果该服务里设定了异常退出后自动重启的话，kill 掉也会被 systemd 再次启动
  • 详细的 systemd 查看以下两篇教程：
    • systemd-tutorial-commands
    • systemd-tutorial-part-two
